Introducción
El Puente de Datos

¿Por qué HTML en un Bootcamp de IA?

A menudo, quienes se enfocan en la Inteligencia Artificial pueden ver el desarrollo web como una disciplina ajena. Sin embargo, este módulo establece la conexión vital: el desarrollo web es la interfaz donde los datos se presentan y se recolectan.

Comprender cómo funciona la estructura de la web permite al especialista en IA realizar tareas de web scraping, limpieza de datos y visualización de resultados de modelos con mucha mayor eficacia.

Pipeline: del HTML de la web al modelo de IA

Idea Central

Todo modelo de IA consume datos. Gran parte de esos datos viven en la web, estructurados en HTML. Quien entiende HTML, entiende de dónde vienen los datos antes de que lleguen al pipeline.

Objetivo: comprender la utilidad transversal del desarrollo web para automatización y gestión de datos en IA
II
Capítulo Dos ⏱ 40 minutos

El Esqueleto
HTML y la Anatomía del DOM

Para construir sobre la web, primero debemos conocer su estructura ósea. Antes de cualquier estilo o lógica, existe un árbol de elementos que define lo que hay en una página y cómo se relacionan entre sí.

El DOM — Document Object Model

El DOM es la representación estructurada del documento HTML. Cada etiqueta se convierte en un nodo con hijos, padres y hermanos. Esta jerarquía es exactamente la que usan BeautifulSoup, Selenium y Playwright.

Ejemplo: extracción de datos con BeautifulSoup

Una vez identificada la estructura en el inspector, la extracción en Python es directa y predecible:

scraper.py
from bs4 import BeautifulSoup
import requests

# 1. Descargar la página
url = "https://books.toscrape.com"
html = requests.get(url).text

# 2. Parsear el DOM
soup = BeautifulSoup(html, "html.parser")

# 3. Navegar el árbol DOM — igual que en el Inspector
#    Inspector nos mostró: article.product_pod > h3 > a[title]
libros = soup.find_all("article", class_="product_pod")

for libro in libros[:3]:
    titulo = libro.find("h3").find("a")["title"]
    precio = libro.find("p", class_="price_color").text
    rating = libro.find("p")["class"][1]  # "Three", "Five"...
    print(f"{titulo[:30]} | {precio} | {rating} estrellas")

# Salida esperada:
# A Light in the Attic      | £51.77 | Three estrellas
# Tipping the Velvet        | £53.74 | One estrellas
# Soumission                | £50.10 | One estrellas

Flujo de trabajo: Inspector de elementos → selector CSS → Python

Conexión con IA — BeautifulSoup

Cuando escribes soup.find('article', class_='product_pod') en Python, estás navegando exactamente el mismo árbol que ves en el inspector. Entender el DOM en el navegador te enseña a escribir mejores selectores en tus scrapers.

Práctica: Abre books.toscrape.com (sitio diseñado para practicar scraping). Usa el Inspector para identificar la etiqueta y clase que contiene el título y el precio de cada libro. Anótalos. Luego escribe el soup.find_all() correspondiente en un notebook de Colab.

Los Mapas
CSS y el Arte de Seleccionar

Una vez comprendida la estructura, aprendemos a dar estilo y, más importante aún, a realizar selecciones precisas. CSS es mucho más que colores y fuentes: es un lenguaje de consulta visual sobre el árbol del DOM.

Anatomía de una regla CSS

Anatomía de una regla CSS — selector + propiedades + valores

Los 5 selectores más útiles para scraping

selectores.css & equivalente Python
/* ── 1. Por etiqueta ──────────────────────── */
h2 { color: navy; }
soup.find_all("h2")

/* ── 2. Por clase ─────────────────────────── */
.price_color { font-weight: bold; }
soup.find_all(class_="price_color")

/* ── 3. Por ID ────────────────────────────── */
#resumen { background: #f0f0f0; }
soup.find(id="resumen")

/* ── 4. Descendiente directo ──────────────── */
ul > li { list-style: none; }
soup.select("ul > li")

/* ── 5. Por atributo ──────────────────────── */
a[href^="https"] { color: green; }
soup.select('a[href^="https"]')

Ejemplo completo: extraer tabla de precios y convertir a DataFrame

extraer_precios.py
from bs4 import BeautifulSoup
import pandas as pd
import requests

html = requests.get("https://books.toscrape.com").text
soup = BeautifulSoup(html, "html.parser")

# Selector CSS directo — copiado del inspector del navegador
datos = []
for item in soup.select("ol.row > li article.product_pod"):
    datos.append({
        "titulo":  item.select_one("h3 > a")["title"],
        "precio":  item.select_one(".price_color").text.strip(),
        "rating":  item.select_one("p.star-rating")["class"][1],
        "stock":   item.select_one(".availability").text.strip()
    })

# DataFrame listo para ML
df = pd.DataFrame(datos)
print(df.head(3))

#    titulo                       precio  rating   stock
# 0  A Light in the Attic         £51.77  Three    In stock
# 1  Tipping the Velvet            £53.74  One      In stock
# 2  Soumission                   £50.10  One      In stock

El selector CSS apunta al card destacado — el mismo sirve en Python

Principio Clave

Los datos en la web rara vez vienen limpios. Los selectores CSS te permiten ignorar el ruido y apuntar exactamente a la señal. Lo que funciona en el inspector, funciona en soup.select().

Práctica: Abre books.toscrape.com en Chrome. Abre la consola del navegador (F12 → Console) y ejecuta document.querySelectorAll('.price_color').length. Verás un número. Luego en Python ejecuta len(soup.select('.price_color')). Ambos deben coincidir exactamente. Esa coherencia es la base del scraping confiable.

Atajos Emmet esenciales para HTML en VS Code

emmet-cheatsheet.txt
── Escribe en VS Code y pulsa Tab ───────────────────────

!                   → boilerplate HTML5 completo
div.contenedor     → <div class="contenedor"></div>
h2#titulo           → <h2 id="titulo"></h2>
ul>li*5             → lista ul con 5 elementos li
table>tr*3>td*4    → tabla 3 filas × 4 columnas
header+main+footer  → 3 semánticas seguidas
p{Texto aquí}      → <p>Texto aquí</p>
span.kpi*3          → 3 spans con clase kpi

── Flujo recomendado ────────────────────────────────────

1. Genera la estructura con Emmet (muy rápido)
2. Añade clases CSS descriptivas (mismas que usarás en soup)
3. Rellena con datos reales de tu modelo
4. Previsualiza con la extensión Live Preview
5. Guarda como portfolio-piece en tu GitHub

Cierre del Módulo

Al terminar este laboratorio habrás creado un documento web completo: estructura semántica, estilos coherentes y métricas reales de un modelo de IA. Las mismas habilidades que usaste para construirlo te servirán para leer, extraer y analizar cualquier documento de la web con Python.

Práctica final: Crea el proyecto desde cero en VS Code. Usa ! + Tab para el boilerplate, añade header, main y footer con Emmet, incorpora métricas reales de cualquier modelo entrenado en el bootcamp, y estiliza la tabla con CSS. El archivo debe abrirse en el navegador y ser comprensible para alguien sin conocimientos técnicos. Súbelo a tu repositorio GitHub como primer artefacto de portfolio web.
