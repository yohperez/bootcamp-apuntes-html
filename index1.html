I
Capítulo Uno ⏱ 10 minutos

Introducción
El Puente de Datos

¿Por qué HTML en un Bootcamp de IA?

A menudo, quienes se enfocan en la Inteligencia Artificial pueden ver el desarrollo web como una disciplina ajena, casi periférica. Sin embargo, este módulo establece una conexión vital: el desarrollo web es la interfaz donde los datos se presentan y se recolectan.

Comprender cómo funciona la estructura de la web permite al especialista en IA realizar tareas de web scraping, limpieza de datos y visualización de resultados de modelos con mucha mayor eficacia. No se trata de convertirse en desarrollador frontend: se trata de dominar el lenguaje en el que hablan los datos en la red.

Idea Central

Todo modelo de IA consume datos. Gran parte de esos datos viven en la web, estructurados en HTML. Quien entiende HTML, entiende de dónde vienen los datos antes de que lleguen al pipeline.

Objetivo: comprender la utilidad transversal del desarrollo web para la automatización y gestión de datos en proyectos de IA
II
Capítulo Dos ⏱ 40 minutos

El Esqueleto
HTML y la Anatomía del DOM

Para construir sobre la web, primero debemos conocer su estructura ósea. Antes de cualquier estilo, cualquier lógica o cualquier modelo, existe un árbol de elementos que define lo que hay en una página y cómo se relacionan entre sí.

El DOM — Document Object Model

El DOM es la representación estructurada del documento HTML. No es el código fuente tal como fue escrito; es el árbol vivo que el navegador construye a partir de ese código. Aprender a navegarlo es fundamental para extraer información específica de cualquier sitio web.

Cada etiqueta HTML se convierte en un nodo. Cada nodo puede tener hijos, padres y hermanos. Esta jerarquía —conocida como árbol del DOM— es la misma que usan herramientas como BeautifulSoup, Selenium o Playwright para automatizar la extracción de datos.

Conexión con IA

Cuando escribes soup.find('div', class_='precio') en Python, estás navegando exactamente este árbol. Entender el DOM en el navegador te enseña a escribir mejores selectores en tus scrapers.

Práctica: Uso del Inspector de Elementos. Aprenderemos a explorar la jerarquía de una página en tiempo real, identificando etiquetas, clases e IDs. Esta es la base técnica para cualquier proyecto de automatización o análisis web. Abriremos una página real, inspeccionaremos su estructura y localizaremos tres elementos distintos usando solo el árbol del DOM.

IV
Capítulo Cuatro ⏱ 20 minutos

Los Mapas
CSS y el Arte de Seleccionar

Una vez comprendida la estructura, aprendemos a dar estilo y, más importante aún, a realizar selecciones precisas. CSS es mucho más que colores y fuentes: es un lenguaje de consulta visual sobre el árbol del DOM.

La Importancia del Selector

CSS no solo sirve para que la web se vea bien; su sintaxis de selectores es la base para apuntar a elementos específicos en los datos. Si puedes seleccionar un elemento con CSS, puedes extraerlo con un script.

Los selectores de CSS —por clase, por ID, por atributo, por posición relativa— son los mismos mecanismos que usa BeautifulSoup con select() o que usa Playwright con page.locator(). Aprender CSS es, en el fondo, aprender a describir patrones dentro de datos estructurados.

Principio Clave

Los datos en la web rara vez vienen limpios. Vienen mezclados con publicidad, menús y decoración. Los selectores CSS te permiten ignorar el ruido y apuntar exactamente a la señal.

Práctica: Selección de elementos por patrones visuales. Identificaremos cómo los estilos visuales nos ayudan a encontrar datos estructurados dentro de una página desordenada. Usaremos el inspector para copiar selectores CSS y probaremos cuántos elementos coinciden con cada patrón antes de escribir una sola línea de Python.

V
Capítulo Cinco ⏱ 30 minutos

Laboratorio Práctico
Trabajamos en VS Code

Todo conocimiento teórico encuentra su verdadero valor en la aplicación. Este capítulo es el cierre del circuito: tomamos lo aprendido sobre estructura y selectores y lo materializamos en un entorno real de trabajo.

Maquetación de un Reporte

El objetivo es construir desde cero una estructura de reporte en VS Code — el entorno de trabajo estándar del desarrollador— aplicando una estructura HTML limpia y dándole forma para presentar los resultados de un proyecto o análisis de forma profesional.

Este tipo de reporte no es solo un ejercicio académico: es exactamente el formato en que se comunican los hallazgos de un modelo de IA a stakeholders no técnicos. Saber construirlo manualmente da la base para generarlo también de forma programática.

Cierre del Módulo

Al terminar este laboratorio habrás creado un documento web completo: estructura semántica, estilos coherentes y contenido real. Las mismas habilidades que usaste para construirlo te servirán para leer, extraer y analizar cualquier documento de la web con Python.

Práctica: Maquetación integral de un reporte técnico utilizando las mejores prácticas de VS Code y estructuración de documentos. Trabajaremos con Emmet para agilizar la escritura de HTML, organizaremos el proyecto en carpetas y produciremos un reporte que podría usarse directamente en un portfolio profesional.
